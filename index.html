DataBase Management System

Q1 : Qu'est ce qu'un DBMS ?

Un systéme de gestion de base de données ou DBMS en abrégé se référe à un logiciel de définition , de stockage , de récupération et de gestion de données de sorte qu'il soit le plus sécurisé possible.



Q2 : Qu'est ce fait un DBMS ?

Le DBMS manage les données ; le moteur de base de données permet aux données d'etre accéder , fermées et modifiées ; Le schéma de base de données définit la structure de la base de données. Le DBMS fournit une vue centralisé des données et qui peuvent etre accessibles par plusieurs utilisateurs , à tout lieu et de maniére controlée. Le DBMS permet aussi d'administrer de maniére uniforme les taches ainsi qu'une sauvegarde des données . Il s'occupe de gérer toutes requetes .



Q3 : Quels sont les composants d'un DBMS ?

Un systéme de gestion de base de données est un systéme sophistiqué qui consiste à un emsemble de composants intégrés qui permet d'obtenir un environnement pour créer , accéder et modifier les bases de données . Cela inclut les composants suivants :
- Le moteur de stockage
- Le catalogue de metadonnées
- Le langage d'accés à la base de données
- Le gestionnaire d'accés et de log
- L'utilitaire de données.



Q4 : Pourquoi apprendre un DBMS ?

Traditionnellement les données sont organisés dans des fichiers. Le DBMS apparait comme un nouveau concept qui à été créer pour surpasser les insuffissances de cette gestion classique. Un DBMS moderne a donc les charactéristiques suivantes :
Des entités du monde réel , des relations basées sur des tables , l'isolation entre les données et l'application , la réduction de la répétition , la consistance et un langage de requetes.
Un systeme de gestion de base de données suit également les critéres de performances suivants :

  • La méthode A.C.I.D (Atomicité , Consistance , Isolation et Durabilité). Elle permet au systeme d'avoir un forme correcte quelque soit l'environnement.
  • Multi-utilisateurs et dégré d'accés : On met en paralléle l'accés et la manipulation des données.
  • Des Vues multiples en fonction des besoins de l'utilisateur
  • Sécurité , l'accés à certaines données est autorisé en fonction du type d'utilisateurs


Q5 : Historique des DBMS ?

Le 1er DBMS a été crée dans le début des années 1960 avec un DBMS navigationnel IBS. Peu de temps aprés IBM développa IMS , un DBMS hiérarchique , qui est toujours utilisé par de grandes compagnies aujourd'hui. En 1971 IDMS est crée grace à la conférence CODASYL. Cela entraina le succes des RDBMS jusqu'a Oracle en 1979. En 1990 des DBMS NoSQL avec l'avénement de l'orienté-objet voir le jour. De nos jours le marché est toujours dominé par les RDBMS cependant les NoSql DBMS continue de gagner en popularité.



Q6 : Types de DBMS ?
  • Dbms hiérarhique

    • Dans un dbms hiérarchique , le modéle de données est organisé comme un arbre. Les données sont stockées de maniére hiérarchique et réprésentées en usant d'une relation parent-enfant. Un parent peut avoir plusieurs enfants mais un enfant a un unique parent.

  • Modéle en réseau

    • Le modéle en réseau octroit la premission à chaque enfant d'avoir plusieurs parents. Il est utile dans la mesure ou il comble le besoin de relations plus complexes comme les relations plusieurs à plusieurs. Dans ce modéle , les entités sont organisés dans un graphique qui est accessible à travers plusieurs chemins.

  • Modéle relationnel

    • Il est le plus utilisé parce qu'il est plus facile d'utilisation. Ce modéle est basé sur la normalisation des données en lignes et colonnes d'une table. les données sont stockées selon une structure fixe et sont manipulées avec SQL.

  • Modéle orienté-objet

    • Dans le modéle orienté-objet , les données sont stockées sous la forme d'objets. La structure appellé classe affiche les données qu'elle contient. Ici , la base de données est défint comme une collection d'objets ou sont stockes toutes les deux les valeurs des données et leurs opérations.



Q7 : SQL vs NoSQL ?
  1. Les bases de données SQL sont relationnelles, NoSQL ne sont pas relationnelles.
  2. Les bases de données SQL utilisent un langage de requête structuré et ont un schéma prédéfini. Les bases de données NoSQL ont des schémas dynamiques pour les données non structurées.
  3. Les bases de données SQL sont évolutives verticalement, les bases de données NoSQL sont évolutives horizontalement.
  4. Les bases de données SQL sont basées sur des tables, tandis que les bases de données NoSQL sont des magasins de documents, de valeurs-clés, de graphiques ou de colonnes larges.
  5. Les bases de données SQL sont meilleures pour les transactions multi-lignes, NoSQL sont meilleures pour les données non structurées comme les documents ou JSON.
